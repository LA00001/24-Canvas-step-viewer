# Анализ мерцания интерактивного вращения — v0.5.08

Контрольный источник: пользовательский лог `inventor_ipt_organizer_v0_5_07_20260619_220128_800.log`.

## Наблюдение

В режиме `Mesh True Front — v0.5.06` статический кадр корректен, но при удержании ЛКМ приложение переключалось в `FastDragWireframe`. Нормальная затенённая поверхность исчезала, появлялся плотный треугольный каркас, а между кадрами наблюдалась белая вспышка/мерцание.

## Измерения v0.5.07

- 62 зарегистрированных `FastDragWireframe`-кадра.
- Время одного drag-кадра: `0.033..0.080 s`.
- Среднее время: около `47.9 ms`.
- Медиана: около `45.5 ms`.
- Активные интервалы между кадрами дают примерно `18..21 FPS`.
- Финальный полный `Mesh True Front`: `0.161..0.198 s`.

## Корневая причина

1. Каждый `MouseMove` немедленно вызывал `Panel.Invalidate()`.
2. Обычный WinForms `Panel` не имел включённого `OptimizedDoubleBuffer`.
3. `Paint` начинался с очистки видимой области белым цветом.
4. После очистки UI-поток синхронно строил до 4608 линий triangle wireframe.
5. При частых `MouseMove` события поступали быстрее, чем завершался рендер, но не объединялись.
6. Полный True Front рендер слишком тяжёлый для перерисовки на каждом движении мыши, поэтому старый код полностью заменял его диагностическим wireframe.

Следовательно, мерцание не связано с Z-buffer, альфа-каналом или геометрией. Это проблема представления кадра и планирования интерактивной перерисовки.

## Исправление v0.5.08

- `BufferedViewerPanel` с `UserPaint`, `AllPaintingInWmPaint`, `OptimizedDoubleBuffer`, `ResizeRedraw`.
- Явный 32-bpp PArgb backbuffer.
- Кадр строится полностью вне экрана и показывается одним atomic swap.
- `OnPaintBackground` не стирает предыдущий кадр между перерисовками.
- MouseMove только обновляет камеру и помечает кадр ожидающим.
- Таймер 30 FPS объединяет накопившиеся движения мыши.
- Во время drag используется текущая ориентация True Front surface в масштабе 55% по ширине и высоте, затем кадр масштабируется до панели.
- В drag-кадре остаётся затенённое непрозрачное тело и лёгкие feature/silhouette edges вместо полного triangle wireframe.
- После отпускания кнопки строится один полный качественный True Front кадр.
- В правом верхнем углу показываются FPS, frame time и количество объединённых событий.

## Новые события журнала

- `DRAG_SESSION_START`
- `DRAG_FRAME_STATS`
- `DRAG_SESSION_SUMMARY`
- `DRAG_CAPTURE_LOST`

Подробные `STEP_ZBUFFER_*` и `STEP_LOCAL_DRAW_SECONDS` подавляются только для высокочастотных промежуточных drag-кадров, чтобы журнал не становился узким местом. Финальный качественный кадр логируется полностью.
